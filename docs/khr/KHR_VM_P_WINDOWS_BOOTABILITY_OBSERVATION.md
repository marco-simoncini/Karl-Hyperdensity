# KHR-VM-P Windows Bootability Observation

Hyperdensity observes the VM-P Windows bootability qualification as a planning signal for Cell/Shell materialization. The observed level is evidence-gated and must not advance beyond `image-inspected`, `firmware-visible`, `bootloader-visible`, `os-boot-signal`, `network-signal`, or `session-ready` without matching Windows evidence.

ResourcePort and ResourceLease remain planning-only for this sprint. `resourceLeaseApply=false` and `resourcePortPersistentLoop=false`.
